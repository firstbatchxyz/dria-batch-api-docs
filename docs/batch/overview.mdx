## Why Batch Inference

Dria’s Batch Inference API is purpose-built for large `.jsonl` workloads where you care about throughput, cost, and predictable delivery windows.

Use it when you need to:

- Run millions of completions without babysitting synchronous endpoints
- Benchmark models offline, including evaluations with strict reproducibility
- Keep inference cost per token as low as possible by aggregating demand
- Pipe experimental agents or RAG pipelines through scheduled backfills

### Architecture at a Glance

1. **Upload** - stream a `.jsonl` file to a pre-signed S3 URL via the `/file/get_upload_url` and `/batch/complete_upload` endpoints.
2. **Queue** - Dria places the file into the shared worker pool, chunking each line into individual inference tasks.
3. **Execute** - workers fan out to the requested models with automatic retries and per-task metadata tracking.
4. **Deliver** - results become available as another `.jsonl` artifact plus the `/batch/all_jobs` and `/batch/job` endpoints for structured retrieval.
