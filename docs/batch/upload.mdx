## Three-Step Upload Flow

1. **Request a pre-signed URL** via `GET /api/v0/file/get_upload_url`.
2. **PUT the file** directly to S3 using the URL payload returns.
3. **Finalize the job** with `POST /api/v0/batch/complete_upload` so workers can start.

Keep the `id` returned from step 1&mdash;it ties the uploaded object to the batch job record.

## Python Example

```python
import os
import requests

FILE_PATH = "batch.jsonl"
API_KEY = os.environ["DRIA_API_KEY"]
BASE_URL = "https://mainnet.dkn.dria.co/api/v0"

headers = {"x-api-key": API_KEY}

# Step 1: Request URL
resp = requests.get(f"{BASE_URL}/file/get_upload_url", headers=headers)
resp.raise_for_status()
payload = resp.json()
url, file_id = payload["url"], payload["id"]

# Step 2: Upload to S3
with open(FILE_PATH, "rb") as handle:
    upload = requests.put(
        url,
        data=handle,
        headers={
            "Content-Type": "binary/octet-stream",
            "Content-Length": str(os.path.getsize(FILE_PATH)),
        },
        timeout=300,
    )
    upload.raise_for_status()

# Step 3: Complete upload
complete = requests.post(
    f"{BASE_URL}/batch/complete_upload",
    headers={**headers, "Content-Type": "application/json"},
    json={"id": file_id},
)
complete.raise_for_status()
print("Batch queued:", complete.json())
```

## CLI Example

```bash
API_KEY=sk-... BASE_URL=https://mainnet.dkn.dria.co/api/v0 FILE=batch.jsonl

UPLOAD=$(curl -sS -H "x-api-key: $API_KEY" "$BASE_URL/file/get_upload_url")
URL=$(echo "$UPLOAD" | jq -r .url)
ID=$(echo "$UPLOAD" | jq -r .id)

curl -sS -T "$FILE" \
  -H "Content-Type: binary/octet-stream" \
  -H "Content-Length: $(wc -c < "$FILE")" \
  "$URL"

curl -sS -X POST -H "x-api-key: $API_KEY" \
  -H "Content-Type: application/json" \
  -d "{\"id\": \"$ID\"}" \
  "$BASE_URL/batch/complete_upload"
```

### Tips

- Uploads expire after a few minutes. If you see `AccessDenied`, fetch a new URL.
- The S3 PUT is anonymous. Do not attach your API key or other headers besides `Content-Type` and `Content-Length`.
- The job does not start until step 3 succeeds; you can safely re-run completion if the response is lost.
- Need more metadata (name, notes, tags)? Pass them in the [complete upload API request](see autogenerated Batch API Reference) under `metadata`.
