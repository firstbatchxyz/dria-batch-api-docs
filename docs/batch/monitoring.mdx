## Check Job Status

Once you complete an upload, poll the job APIs until the status transitions to `completed` or `failed`.

```python
import requests, time

BASE = "https://mainnet.dkn.dria.co/api/v0"
API_KEY = "sk-..."
headers = {"x-api-key": API_KEY}

job_id = "<returned-from-complete-upload>"
while True:
    resp = requests.get(f"{BASE}/batch/job", params={"id": job_id}, headers=headers)
    resp.raise_for_status()
    job = resp.json()
    print(f"{job['status']} | processed={job['processed']} / {job['total_tasks']}")
    if job["status"] in {"completed", "failed"}:
        break
    time.sleep(10)
```

Useful fields returned by `/batch/job`:

- `status`: `queued`, `processing`, `completed`, or `failed`
- `processed`: count of tasks with terminal states
- `failed_tasks`: aggregated failures (network, model, validation)
- `result_url`: signed URL for the output `.jsonl`

Call `/batch/all_jobs` to list the most recent jobs for a key. When you need full schema documentation, jump to the autogenerated **Batch API Reference**.

## Retrieve Outputs

Each line in the result file mirrors the request `custom_id` and includes model response, token accounting, and any error payloads. Use a streaming reader to avoid loading everything into memory:

```python
import json, requests

result_url = job["result_url"]  # returned from /batch/job
resp = requests.get(result_url, stream=True, timeout=300)
for line in resp.iter_lines():
    record = json.loads(line)
    if "error" in record:
        print("Failed", record["custom_id"], record["error"]["message"])
    else:
        print(record["custom_id"], record["result"][:80])
```
