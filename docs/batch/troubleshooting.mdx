## Common Issues

| Symptom | Likely Cause | Fix |
| --- | --- | --- |
| `AccessDenied` during upload | Pre-signed URL expired | Request a new URL and retry the PUT. URLs expire within minutes. |
| `400 invalid body` when completing upload | Missing `id` or malformed JSON | Ensure you send `{"id": "<value>"}` with `Content-Type: application/json`. |
| Job stuck in `queued` | Quota exhausted or workers saturated | Check [limits](docs/batch/limits). Contact support if you need a temporary bump. |
| `failed_tasks` > 0 | Model limit hit, prompt too large, or request invalid | Inspect the output `.jsonl` to see per-task error payloads. |
| Result URL returns 404 | Artifact expired | Download results within 7 days or contact support for rehydration. |

## Example Error Handling

```python
def require_ok(resp):
    if resp.ok:
        return resp.json()
    raise RuntimeError(f"{resp.request.method} {resp.request.url} failed ({resp.status_code}): {resp.text}")

upload_meta = require_ok(requests.get(f"{BASE}/file/get_upload_url", headers=headers))
```

## FAQ

- **How do I monitor jobs?** Poll `/batch/job` or use the [web interface](docs/batch/web-interface). Both read from the same data store.
- **How are results delivered?** Each job emits a signed URL plus metadata through the API. Download the `.jsonl` and join rows using `custom_id`.
- **Can I restart a failed job?** Re-upload the file (even with the same contents) to create a new job. Failed tasks are reported, not retried indefinitely.
- **Where do I find exact schemas?** Every property and enum is documented in the autogenerated **Batch API Reference** beneath this section.
- **I need a bigger limit.** Email inference@dria.co with the plan, expected concurrency, and time window.

Still blocked? Mention the job `id` when emailing support so the team can trace logs quickly.
