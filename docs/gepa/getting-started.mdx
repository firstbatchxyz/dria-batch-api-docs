## Prerequisites

- Dria API key with GEPA access
- Dataset of labeled examples (JSON serializable)
- Familiarity with the models listed in [GEPA Resources](docs/gepa/resources)

## Build the Payload

Every execution is described by a single JSON object:

| Field | Description |
| --- | --- |
| `customId` | Friendly identifier for the run. Appears in logs. |
| `strategy` | Currently `RPM` (Reflective Prompt Mutation). |
| `model` | The model GEPA should use for generations. |
| `datasetColumns` | Ordered list of variables referenced in your prompt template. |
| `budget` | Number of inference calls you are willing to spend. Minimum 10; 25-50 recommended. |
| `minibatchSize` / `paretoSize` | Split of dataset rows for feedback vs validation. Must add up to dataset length. |
| `evaluator` | JSON string describing the evaluator (`model`, `metric`, thresholds). |
| `dataset` | Array of `{ "input": {...}, "expectedOutput": "..." }`. Order matters (minibatch rows first). |
| `prompt` | Seed prompt template that references `datasetColumns`. |

**Dataset sizing rule:** `len(dataset) == minibatchSize + paretoSize`. GEPA reads the first `minibatchSize` examples for feedback and the remainder for Pareto validation.

## Minimal Example

```json
{
  "customId": "demo-support",
  "strategy": "RPM",
  "model": "gpt-4o",
  "datasetColumns": ["customer_name", "product", "issue"],
  "budget": 25,
  "minibatchSize": 2,
  "paretoSize": 2,
  "evaluator": "{\"model\": \"gpt-4o-mini\", \"metric\": \"exact_match\", \"threshold\": 0.8}",
  "dataset": [
    {
      "input": {"customer_name": "Rory", "product": "Router", "issue": "packet loss"},
      "expectedOutput": "Rory, QoS has been tuned to eliminate packet loss."
    },
    {
      "input": {"customer_name": "Amina", "product": "Router", "issue": "needs parental controls"},
      "expectedOutput": "Amina, parental controls are enabled with a weekend preset."
    },
    {
      "input": {"customer_name": "Leo", "product": "Router", "issue": "firmware question"},
      "expectedOutput": "Leo, firmware 3.2.4 deployed with rollback ready."
    },
    {
      "input": {"customer_name": "Yara", "product": "Router", "issue": "coverage gaps"},
      "expectedOutput": "Yara, mesh nodes repositioned to close the gaps."
    }
  ],
  "prompt": "Write a short status update for {customer_name} about their {product}: {issue}."
}
```

## Run, Monitor, Retrieve

1. `POST /api/v0/gepa/start_execution` with your payload. Response includes `evolutionId`.
2. Poll `GET /api/v0/gepa/get_single_execution/{evolutionId}` every 10-30 seconds. Status values: `pending`, `in_progress`, `completed`, `failed`.
3. Once completed, fetch generations via `GET /api/v0/gepa/get_execution_prompts`.

Look up the precise request/response schemas in the autogenerated **GEPA API Reference** for this section.
