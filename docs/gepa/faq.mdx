### How long does an execution take?

Depends on the `budget`, model latency, and dataset size. A typical `budget=30` run on `gpt-4o` finishes in 5-15 minutes. Smaller budgets finish faster but may converge to lower scores.

### What is the minimum dataset size?

`len(dataset) = minibatchSize + paretoSize`. The minimum that makes sense is 4 rows (2 feedback + 2 pareto), but most users see better results with 6-8 rows. Order matters: feedback rows must appear first.

### Can I cancel an execution?

Executions run until they hit the budget or encounter a fatal error. Monitor via `GET /gepa/get_single_execution/{evolutionId}` and simply ignore the run if you start a replacement. Official cancellation is on the roadmap.

### Why did my execution fail?

Common causes:

- Dataset size mismatch
- Unsupported `model`
- Evaluator JSON missing the `model` field or improperly escaped
- Missing `input`/`expectedOutput` keys
- `expectedOutput` is an empty string

Check the error message returned by `get_single_execution` and compare your payload with the examples in this section or the **GEPA API Reference**.

### Which models can I target?

See [Resources](docs/gepa/resources) for the live list. All payloads must reference models from that list for both `model` and `evaluator`.

### Which strategy should I choose?

Currently `RPM` (Reflective Prompt Mutation) is the supported strategy and works for instruction-following, evaluations, and summarization scenarios. Additional strategies will appear as separate enum values in the API reference.

### Do I really need a model in the evaluator?

Yes. The evaluator runs real model calls to automatically score generations. Provide a separate `model` (usually a lighter one like `gpt-4o-mini`) plus metric configuration.
