## Supported Models

GEPA can target any of the following generation models:

- `gpt-5-2025-08-07` and `gpt-5-mini-2025-08-07`
- `gpt-4.1-2025-04-14`
- `gpt-4o-2024-08-06`
- `gpt-4o-mini-2024-07-18`

Evaluator payloads can use the same set or a cheaper variant (for example, run generations on `gpt-4o` but evaluate with `gpt-4o-mini`).

## Evaluator Cheatsheet

| Metric | When to use | Notes |
| --- | --- | --- |
| `exact_match` | Deterministic answers (math, structured outputs) | Add `partialCredit` for near matches. |
| `coherence` | Narrative or explanatory responses | Pair with a higher `threshold` to avoid verbose fluff. |
| `recall` | Summaries that must include specific facts | Great for knowledge-base distillation. |

Remember to encode the evaluator JSON as a string in the payload (see [Getting Started](docs/gepa/getting-started)).
